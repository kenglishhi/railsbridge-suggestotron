What does it all mean?

Ruby

is a programming language, similar to Python and Perl.

  • dynamically typed (no need for “int i”),
  • interpreted
  • It has dozens of shortcuts
  • RegEx support and works well for shell scripting.

Rails

is a Ruby Gem, or a programming library.

  • helps make Model View Controller (MVC) web applications
  • classes for saving to the database,
  • handling URLs and displaying html

    • IRB

    interactive Ruby console for live debugging(type “irb” to use). Rails has a special console 

     
     rails console
     rails c # shortcut

    Tips for people new to Ruby

    1) Ruby lets you be concise 

    10.times do  
  puts "Hello World" 
end

    2) Everything is an Object

    Everything in Ruby that can be bound to a variable name is a full-fledged object.

    This has interesting consequences

    Classes are Objects!

    Array is a constant name that is bound to the Array class object.

    Creating new objects does not require special syntax. We just send new to the class object. 

    a = Array.new

    No primitives (like in Java)

    Even integers are full fledged objects.

      0.zero?    # => true
  1.zero?    # => false
  1.abs      # => 1
  -1.abs     # => 1
  1.methods  # => list of methods for object 1
  2.+(3)     # => 5  (same as 2+3)
  10.class   # => Fixnum
  (10**100).class # => Bignum

    3) Ruby is waaaaay dynamic

    Methods can be added to classes at any point ... even built in classes. 

    class Integer
  def even?
    (self % 2) == 0
  end
end

p (1..10).select { |n| n.even? }
  # => [2, 4, 6, 8, 10]

    Use Ruby Convention

    ClassNames

    method_names and variable_names

    methods_asking_a_question?

    @instance_variables

    $global_variables

    SOME_CONSTANTS or OtherConstants

    4) Code blocks are EVERYWHERE, learn to love them

    Bad code (maybe what you are used to) : 

    for i in (1..10)
  puts i
end
    Ruby Way: 
    (1..10).each do |i |
  puts i
end
    Or all on one line! 
    (1..10).each { |i| puts i }
